# Confirmation Saints

> *Help confirmation candidates discover and choose their patron saint.*

[![App Store](https://img.shields.io/badge/Download-App%20Store-0D96F6?logo=apple)](https://apps.apple.com/app/confirmation-saints/id6762463641)

A SwiftUI iOS app (iPhone + iPad) that helps Catholic Confirmation candidates — primarily teenagers, but also adults — explore the lives of the saints and find a meaningful patron for their Confirmation journey.

**🎉 Status:** iOS app **v1.0.0 is live on the App Store** — [apps.apple.com/app/confirmation-saints](https://apps.apple.com/app/confirmation-saints/id6762463641). Android port is next; see [Android Port Guide](#-android-port-guide) below.

**✨ What's New:** Expanding our collection with additional curated saints — the app now features **over 100 Catholic saints** with rich biographies, quotes, feast days, and more to help you find the perfect patron for your Confirmation.

## 🎯 Target Audience

- **Primary:** Teenagers preparing for the Sacrament of Confirmation
- **Secondary:** Adults receiving Confirmation later in life
- **Also useful for:** Catechists, youth ministers, RCIA leaders

## 📱 Features (shipped in v1.0.0)

- **Welcome / Onboarding** — 4-page first-launch walkthrough (replayable from Settings)
- **About Confirmation** — In-app explanation of the sacrament and patron-saint tradition
- **Explore by Category** — Browse saints grouped by: *Patron Of*, *Interests & Affinities*, *Age*, *Region of Origin*, *Life State*, *Century / Era*, *Gender*
- **Saints List with Search** — Diacritic-insensitive search across name, patronage, affinities, tags, and country (via shared `String+Diacritics` extension)
- **Filter Chips** — Quick filters for region, life state, age category, gender, affinity
- **Saint Detail** — Image + attribution, quote, "Why a Confirmation Saint?", biography, feast day, birth/death/canonization dates, patronage, tags, affinities, and source links
- **Bilingual** — Full English and Spanish (UI strings + saint content). Language auto-detects from the system locale and is switchable from Settings.
- **Over 100 saints** curated, with public-domain imagery from Wikimedia Commons
- **Offline-first** — All content bundled; no network required

### iOS Pattern: MVVM + SwiftUI

- **Models** (`Models/Saint.swift`, `Models/Category.swift`) — `Codable`, `Sendable` structs
- **ViewModel** (`ViewModels/SaintListViewModel.swift`) — `@Observable` (iOS 17 macro), `@MainActor`; owns loaded saints/categories/sections and all filter state; exposes `filteredSaints` and `saints(forCategoryGroup:valueId:)`
- **Service** (`Services/SaintDataService.swift`) — singleton, `@MainActor`; loads per-language JSON from the bundled `SharedContent/` folder reference
- **Views** — SwiftUI; entry is `CatholicSaintsApp` → `SplashView` overlay + (`WelcomeView` if first launch else `ContentView` with a 4-tab `TabView`: About / Explore / Saints / Settings)
- **State persistence** — `@AppStorage("appLanguage")`, `@AppStorage("hasSeenWelcome")`

### Localization Strategy (important — this changed)

Two parallel mechanisms:

| Content | Mechanism | Location |
|---|---|---|
| UI strings (buttons, labels) | Apple String Catalog | `ios/CatholicSaints/Resources/Localizable.xcstrings` |
| Saint / category / info content | **Per-language JSON files** (not in-file `LocalizedText` objects) | `SharedContent/**/*-{en,es}.json` |

Each saint file (`saints-en.json`, `saints-es.json`) contains **plain-string** fields translated into that language. Matching/filtering logic (e.g. filtering by `patronOf: "missionaries"`) runs against **canonical English identifier values**, which are present in both files. Localized display equivalents live in optional `display*` arrays (`displayPatronOf`, `displayTags`, `displayAffinities`) alongside them — this is what the UI renders, while filters compare against the canonical values.

> **Why this matters for Android:** same logic applies. Filter on `patronOf` / `affinities` / `tags` (English canonical ids), render from `display*` (localized labels).

### Data Model (Saint schema)

Canonical schema for each object in `saints-{lang}.json → saints[]`:

```jsonc
{
  "id": "therese-of-lisieux",            // stable kebab-case id; also the image filename stem
  "name": "St. Thérèse of Lisieux",      // localized display name
  "feastDay": "10-01",                   // MM-DD (formatted to "October 1" in-app)
  "birthDate": "1873-01-02",             // ISO date or null
  "deathDate": "1897-09-30",             // ISO date or null
  "canonizationDate": "1925-05-17",      // ISO date or null (null for pre-congregation saints)
  "country": "France",                   // localized country name
  "region": "Europe",                    // one of: Europe | Americas | Africa | Asia | Middle East (English canonical)
  "gender": "female",                    // "male" | "female" (canonical)
  "lifeState": "religious",              // "religious" | "lay" | "married" | "clergy" (canonical)
  "ageCategory": "young",                // "young" | "adult" (canonical)
  "patronOf": ["missionaries", "florists", "pilots", "France"],  // English canonical ids used for matching
  "displayPatronOf": ["misioneros", ...],                         // localized labels (ES file only; optional)
  "tags": ["young", "writer", "mystic", "Doctor of the Church"],
  "displayTags": [...],
  "affinities": ["reading", "writing", "nature", "prayer"],
  "displayAffinities": [...],
  "quote": "Miss no single opportunity...",
  "biography": "Marie Françoise-Thérèse Martin was born...",
  "whyConfirmationSaint": "She shows that holiness...",
  "image": { "filename": "therese-of-lisieux.jpg", "attribution": "Public domain, via Wikimedia Commons" },
  "sources": ["Franciscan Media", "Catholic News Agency"],
  "sourceURLs": { "Franciscan Media": "https://...", "Catholic News Agency": "https://..." }
}
```

Wrapper envelope (all three JSON kinds share this shape):

```jsonc
{ "version": "1.0.0", "language": "en", "lastUpdated": "YYYY-MM-DD", "saints": [ ... ] }
```

### Category Model

`categories-{lang}.json → categories[]` is a list of **groups**; each group has `values[]` with `{ id, label }`. Current groups (v1.0.0):

| Group id | Values | Matches against |
|---|---:|---|
| `patronage` | 18 | `saint.patronOf` (contains, diacritic-insensitive) |
| `interests` | 14 | `saint.affinities` ∪ `saint.tags` |
| `age-category` | 2 | `saint.ageCategory` |
| `region` | 5 | `saint.region` |
| `life-state` | 4 | `saint.lifeState` |
| `era` | 5 | derived from `birthDate` / `deathDate` |
| `gender` | 2 | `saint.gender` |

The group → field mapping is implemented in `SaintListViewModel.saints(forCategoryGroup:valueId:)`. Value ids with hyphens are un-hyphenated for matching (e.g. `young-adults` ↔ `young adults`).

### Key Technical Choices

- **iOS 17.0** deployment target (NavigationStack, `@Observable` macro, searchable)
- **Swift 6.0**, strict concurrency: models are `Sendable`, services are `@MainActor`
- **XcodeGen** drives `CatholicSaints.xcodeproj` from `ios/project.yml` — never edit the `.pbxproj` directly
- **Display name:** "Confirmation Saints" (`INFOPLIST_KEY_CFBundleDisplayName`, `PRODUCT_NAME`)
- **Bundle ID:** `com.jorgebalderas.ConfirmationSaints`
- **App icon** generated programmatically by `_generate_icon.py` (single 1024×1024 PNG, Xcode derives the rest)
- **Diacritic-insensitive search** everywhere, via `String+Diacritics.swift` (`containsIgnoringDiacritics`, `equalsIgnoringDiacritics`)

## ➕ How to Add a New Saint

1. Pick a stable kebab-case `id` (this is also the image filename stem).
2. Add a parallel entry to **both** `SharedContent/saints/saints-en.json` and `saints-es.json`, following the [Saint schema](#data-model-saint-schema) above. Keep the canonical (English) values in `patronOf` / `tags` / `affinities` / `region` / `lifeState` / `ageCategory` / `gender` **identical** in both files; translate only `name`, `country`, `biography`, `quote`, `whyConfirmationSaint`, and the `display*` arrays.
3. Drop an image at `SharedContent/images/{id}.jpg` (Wikimedia Commons public domain, ~400px wide) and set `image.filename` + `image.attribution`.
4. Run `cd ios && xcodegen generate` only if you added new top-level folders; otherwise just build — the folder reference picks up the new JSON automatically.

`_generate_saints.py` and `_download_saint_images.py` can scaffold entries and fetch images in bulk.

## 🌍 How to Add a New Language

1. **UI strings** — open `ios/CatholicSaints/Resources/Localizable.xcstrings` in Xcode and add the new language; translate each entry.
2. **Saint/category/info content** — create parallel files: `saints-{lang}.json`, `categories-{lang}.json`, `confirmation-info-{lang}.json`. Copy the English file as a template and translate the localizable fields; keep all canonical/id fields unchanged.
3. Wire the language code into `LocalizationService` / `appLanguage` handling as needed.

## 🤖 Android Port Guide

The repo is structured so an Android (Kotlin + Jetpack Compose) app can consume `SharedContent/` directly.

### Reusable as-is

- ✅ **All of `SharedContent/`** — `saints-{en,es}.json`, `categories-{en,es}.json`, `confirmation-info-{en,es}.json`, and `images/*.jpg`. This is the **canonical cross-platform data source**; do not fork it.
- ✅ **Data schema** — see [Data Model](#data-model-saint-schema). Model it as Kotlin `@Serializable` data classes.
- ✅ **Category / filter matching logic** — mirror `SaintListViewModel.saints(forCategoryGroup:valueId:)`. Match against canonical English ids (`patronOf`, `affinities`, `tags`, `region`, `lifeState`, `ageCategory`, `gender`), **not** the `display*` arrays.
- ✅ **Diacritic-insensitive search** — on Android use `java.text.Normalizer.normalize(s, NFD)` + `\\p{InCombiningDiacriticalMarks}` stripping, or a `Collator` with `Collator.SECONDARY` strength. Behavior must match iOS (see `String+Diacritics.swift`).
- ✅ **Era derivation** (category group `era`) — reuse the date ranges from `SaintListViewModel.matchesEra`.
- ✅ **Source URL audit** — the curated `sourceURLs` map is known-good; do not regenerate.
- ✅ **App name and branding** — "Confirmation Saints" (display name), purple/gold liturgical palette, Chi-Rho icon.

### Needs Android-native reimplementation

- 🔄 **SwiftUI views** → Jetpack Compose (`NavigationStack` → `NavHost`; `TabView` → `NavigationBar`; `@Observable` VM → `ViewModel` + `StateFlow`)
- 🔄 **`@AppStorage`** → `DataStore<Preferences>` (keys to preserve: `appLanguage`, `hasSeenWelcome`)
- 🔄 **`Localizable.xcstrings`** → Android `res/values*/strings.xml` (one `values-es/` etc. per language)
- 🔄 **JSON loading** — bundle `SharedContent/` via a Gradle source-set include, read from `assets/`
- 🔄 **App icon** — `_generate_icon.py` outputs a single 1024×1024 PNG; Android needs adaptive-icon foreground/background layers (regenerate or adapt the Pillow script)
- 🔄 **Bundle identifier** — suggested `com.jorgebalderas.confirmationsaints` (lowercased per Android convention)

### Decisions Android must respect

- **Canonical ids are English.** Both language files share identical values in `patronOf`, `tags`, `affinities`, `region`, `lifeState`, `ageCategory`, `gender`. Filter against these; render from `display*`.
- **Welcome screen gating** by a persisted boolean (`hasSeenWelcome`), replayable from Settings.
- **Language auto-detect from locale** on first launch (Spanish → `es`, otherwise `en`), then user-overrideable.
- **Feast day format is `MM-DD`** (no year). Formatting to "October 1" / "1 de octubre" is a UI concern.
- **Display name** is "Confirmation Saints" — the internal folder/module can be named whatever fits Android conventions.

See `android/README.md` for the current port status.
